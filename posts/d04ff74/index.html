How to Set Up a Web Server in Windows 7

Overview

为了模拟 OpenWrt 的固件升级或者 opkg 软件包升级，我需要搭建一个 web server 存放静态文件。本文使用微软自带的 Internet Information Services (IIS)，搭建一个文件共享站点，支持 http 协议下载文件。

Procedure

Step 1: 开启 IIS 服务

单击 “开始” 图标，选择 “控制面板”，单击 “程序”，单击程序和功能的 “打开或关闭 Windows 功能”，选择 “Internet 信息服务” 可选框，然后点击确定。

Step 2: IIS 管理

单击 “开始” 图标，在搜索框输入 “IIS”，然后单击 “Internet 信息服务(IIS)管理器”。

Step 3: 站点配置

a. 点击左上角的小三角展开 website 配置，如图：

b. 展开 “网站”
c. 右击 “Default Web Site”，选择 “添加虚拟目录”
d. 填写别名和映射的物理路径，然后点击确定，就会在 “Default Web Site” 添加以别名为名称的子级

Step 4: 添加虚拟目录

给新添加的虚拟目录添加 IUSR 组分配可读权限，以别名为 “lede” 为例。

a. 右击 “lede”(新添加的虚拟目录的别名)
b. 单击 “编辑权限”，在弹窗找到 “安全” 并选择
c. 点击 “编辑”，再点击 “添加”，再点击 “高级(A)…”，点击 “立即查找(N)”，在列表找到 “IUSR”，选择并确定，有 “应用” 按键的先点击应用再确定。

Step 5: 启用匿名访问

单击 “lede”，在侧边栏找到 “身份验证”，双击它，如果匿名验证状态是禁用，点击右上角的 “启用”。

Step 6: MIME 类型

a. 单击侧边栏的 “Default Web Site”
b. 双击 “MIME 类型”，点击右上角的 “添加”，文件扩展名填入 “.sig”，MIME 类型填入 “application/octet-stream”，点击 “确定”
c. 重复步骤 b，添加扩展名 “.”，MIME 类型为 “application/octet-stream”
d. 重复步骤 b，添加扩展名 “.ipk”，MIME 类型为 “file/download”

这里根据自己的需要添加，IIS 默认关联了一些类型的静态文件，我们只需要添加里面没有我们需要的扩展名，比如我搭建的 web server 用于存储 OpenWrt 的 ipk 软件包，用于测试，所以我添加了 .ipk 和 .sig，以及没有扩展名的类型。
